🌐 ONI Developer Kit

Build and deploy websites on the decentralized ONI network.

This developer kit provides everything you need to create, host, and manage .orb websites on the Orbital Network Infrastructure.

📋 Overview

ONI (Orbital Network Infrastructure) is a fully decentralized peer-to-peer internet network. It replaces traditional web infrastructure (DNS servers, hosting providers, centralized registrars) with a distributed network where users host and resolve content themselves.

ONI NETWORK OVERVIEW ┌─────────────┐ ┌─────────────┐ ┌─────────────┐ │ User A │ ←──→ │ User B │ ←──→ │ User C │ │ Browser │ │ Browser │ │ Browser │ │ + Node │ │ + Node │ │ + Node │ │ + Hosting │ │ + Hosting │ │ + Hosting │ └─────────────┘ └─────────────┘ └─────────────┘ │ │ │ └────────────────────┼────────────────────┘ │ ▼ ╔══════════════════════════╗ ║ ONI P2P Network ║ ║ WebSocket / WebRTC ║ ║ Decentralized DNS ║ ╚══════════════════════════╝

✨ Key Features for Developers

  • Free .orb domains — No registration costs, no renewal fees
  • Self-hosting — Host directly from your own machine or server
  • Docker support — Deploy with a single docker-compose command
  • P2P sync — Content automatically syncs across the network
  • Standard HTML5 — Build with HTML, CSS, JS — no special framework needed
  • Domain linking — Link multiple domains to the same site

🚀 Quick Start

1

Install ONI

curl -fsSL https://raw.githubusercontent.com/technicdev/ONI/main/install.sh | bash

Or clone the repository:

git clone https://github.com/technicdev/ONI.git ~/ONI && cd ~/ONI && bash install.sh --no-clone
2

Start the ONI Manager

oni manager

Or directly:

python3 ~/ONI/apps/oni_manager.py
3

Register a .orb Domain

In the ONI Manager, go to the Domains tab, enter your domain name, and click Register Domain.

Or use the web registrar at http://127.0.0.1:8080

4

Host Your Website

Create an HTML file and host it:

mkdir -p ~/mysite && echo "

Hello ONI!

" > ~/mysite/index.html
oni node --host-domain mysite.orb ~/mysite
5

Browse Your Site

Launch the Orbit Browser and navigate to:

oni browser --url orb://mysite.orb

Or open the browser app and type orb://mysite.orb in the address bar.

Available TLDs

Domain Rules

  • Domain names: 1-63 characters
  • Allowed characters: a-z, 0-9, hyphens (-)
  • No hyphens at start or end
  • Domains are case-insensitive (stored lowercase)
  • Domains are FREE to register — no costs, no renewals
Note: .orb domains are not traditional DNS domains. They only work within the ONI network via the Orbit Browser or ONI-compatible tools.

📂 Hosting a Site

Creating Your Website

ONI supports standard HTML5 websites. Create your files just like you would for any web server:

mysite.orb/
├── index.html
├── style.css
├── script.js
├── images/
│   └── logo.png
└── pages/
    ├── about.html
    └── contact.html

Hosting via ONI Manager

  1. Open the ONI Manager app
  2. Go to the Hosting tab
  3. Enter your domain (e.g., mysite.orb)
  4. Browse to your website directory
  5. Click Host Website

Hosting via CLI

oni node --host-domain mysite.orb /path/to/mysite.orb

Or directly with Python:

python3 ~/ONI/p2p/oni_node.py --host-domain mysite.orb /path/to/mysite.orb

Hosting Multiple Domains

oni node --host-domain site1.orb /path/to/site1 --host-domain site2.orb /path/to/site2

🔗 Domain Linking

Domain linking allows you to point multiple .orb domains to the same website content. This is useful for:

  • Having multiple TLDs for the same site (e.g., mysite.orb and mysite.orb.dev)
  • Creating redirects or aliases
  • Setting up subdomains

How Domain Linking Works

When you host a site under one domain, you can create DNS records that point other domains to the same content. The ONS system resolves the linked domain to the hosting node.

DOMAIN LINKING ARCHITECTURE User types: DNS Resolution: orb://mysite.orb ──────→ ONS ──────→ Node A (hosting) │ orb://mysite.orb.dev ──→ ONS ──┘ │ orb://mysite.orb.fun ──→ ONS ──┘ (All resolve to same content)

How to Link Domains

1

Register All Domains

Register each domain you want to link (e.g., mysite.orb, mysite.orb.dev, mysite.orb.fun)

2

Host Your Content Under the Primary Domain

oni node --host-domain mysite.orb /path/to/site
3

Create DNS Records for Linked Domains

Use the ONS API or ONI Manager to create CNAME-like records:

curl -X POST http://127.0.0.1:5353/link \ -H "Content-Type: application/json" \ -d '{"domain": "mysite.orb.dev", "target": "mysite.orb"}'
4

Verify Linking

curl http://127.0.0.1:5353/resolve/mysite.orb.dev

When users visit orb://mysite.orb.dev, they'll see the same content as orb://mysite.orb.

Pro Tip: You can also use the ONI Manager's Domains tab to manage linked domains through the GUI.

🖥️ Self-Hosting

Self-hosting means running ONI on your own hardware to serve your .orb website directly from your machine. No third-party hosting provider needed.

Self-Hosting Options

🖥️ Option 1: Local Machine (Development)

Run directly on your computer for development and testing.

Pros:

  • Easy to set up
  • No additional hardware needed
  • Immediate changes

Cons:

  • Only available when your computer is on
  • Limited to your network

🐳 Option 2: Docker Container (Production)

Run ONI in a Docker container for reliable, always-on hosting.

Pros:

  • Runs 24/7
  • Easy to deploy and update
  • Isolated environment
  • Portable across servers

Cons:

  • Requires Docker knowledge
  • Needs a server or VPS

☁️ Option 3: VPS/Cloud Server

Run ONI on a $5/month VPS from any provider (DigitalOcean, Linode, etc.).

Pros:

  • Always online
  • Static IP address (important for peer discovery)
  • Low cost

Cons:

  • Requires basic server administration

🐳 Docker Deployment

docker-compose.yml

The docker-compose.yml file defines all ONI services. Here's the complete configuration:

Prerequisites: Docker and Docker Compose must be installed on your system.
Install Docker: curl -fsSL https://get.docker.com | sh 
version: '3.8'

services:
  # ===== ONI P2P Node =====
  oni-node:
    build: .
    container_name: oni-node
    ports:
      - "6881:6881"
    volumes:
      - ./data:/app/data
      - ./sites:/app/sites
    environment:
      - ONI_NODE_PORT=6881
      - ONI_HOST=0.0.0.0
    command: python3 p2p/oni_node.py --host 0.0.0.0 --port 6881
    restart: unless-stopped
    networks:
      - oni-network

  # ===== ONS Server =====
  ons-server:
    build: .
    container_name: oni-ons
    ports:
      - "5353:5353"
    volumes:
      - ./data:/app/data
    environment:
      - ONS_PORT=5353
    command: python3 ons/ons_server.py --port 5353
    restart: unless-stopped
    networks:
      - oni-network

  # ===== Domain Registrar =====
  oni-registrar:
    build: .
    container_name: oni-registrar
    ports:
      - "8080:8080"
    volumes:
      - ./data:/app/data
    environment:
      - REGISTRAR_PORT=8080
    command: python3 orbit-registrar/registrar.py --port 8080
    restart: unless-stopped
    networks:
      - oni-network

  # ===== Orbit Browser (Web Mode) =====
  orbit-browser:
    build: .
    container_name: orbit-browser
    ports:
      - "9090:9090"
    volumes:
      - ./data:/app/data
    environment:
      - BROWSER_PORT=9090
    command: python3 apps/orbit_browser_app.py --port 9090 --no-gui
    restart: unless-stopped
    networks:
      - oni-network
    depends_on:
      - oni-node
      - ons-server

networks:
  oni-network:
    driver: bridge

Dockerfile

FROM python:3.11-slim

WORKDIR /app

# Install dependencies
RUN pip3 install --no-cache-dir websockets flask requests aiohttp

# Copy ONI source
COPY . .

RUN mkdir -p data/{cache,domains,peers,logs} sites

# Expose ports

🔄 P2P Sync

The ONI network uses peer-to-peer synchronization to share content between users. When two or more users run ONI nodes, their systems automatically share:

  • Domain registrations — Who owns which .orb domains
  • Website content — Hosted site files
  • Peer lists — Who's currently online
  • DNS records — Domain resolution information

How Sync Works

P2P SYNC FLOW User A User B │ │ │ ─── Announce self ───→ │ │ ←── Peer list ─────── │ │ │ │ ─── Share domains ───→ │ │ ←── Share content ─── │ │ │ │ ─── Sync bookmarks ──→ │ │ ←── Sync history ──── │ │ │ │ Both have the same │ │ websites and data │

Automatic Sync

The ONI node automatically:

  1. Announces itself to the network every 30 seconds
  2. Responds to peer discovery requests
  3. Shares hosted domain information
  4. Forwards content requests to peers hosting the requested domain

Manual Sync (Bookmarks & History)

In the Orbit Browser, use Tools → Sync with Device to manually sync bookmarks and history with another ONI user:

oni browser # Then go to Tools > Sync with Device

Enter the other user's IP address and port to initiate sync.

Cross-Device Data Sharing

When User A and User B both have ONI running:

  • User A registers mysite.orb and hosts content
  • User B's node discovers User A's node through the network
  • When User B visits orb://mysite.orb, their browser fetches the content from User A's node
  • Both users see the same website with the same content
Pro Tip: For the best sync experience, ensure both users have their ONI nodes running on ports that are reachable from each other's networks. For internet-wide sync, each node needs a public IP or port forwarding.

🧭 ONS System

The Orbit Name System (ONS) is the decentralized DNS for .orb domains. It replaces traditional DNS with a P2P-based resolution system.

How ONS Works

  1. When a domain is registered, it's stored in the ONS registry
  2. The registry is distributed across all ONI nodes
  3. When a user navigates to orb://mysite.orb, the browser queries ONS
  4. ONS returns the hosting node's address
  5. The browser fetches content from that node

👥 Peer Management

Peers are other ONI nodes on the network. Managing peers helps ensure your content is discoverable and you can access other users' sites.

Viewing Peers

In the ONI Manager, go to the Peers tab to see connected nodes.

Via API:

curl http://127.0.0.1:5353/peers

Connecting to Peers

Peers are discovered automatically through:

  • DHT (Distributed Hash Table) — Nodes find each other through the network
  • Manual connection — Specify a peer's address directly
  • Bootstrap nodes — Initial connection points for new nodes

Peer Status Codes

StatusDescription
🟢 OnlinePeer is connected and responding
🟡 DegradedPeer is reachable but slow
🔴 OfflinePeer is not responding
⚪ UnknownPeer discovered but not yet connected

🔧 Troubleshooting

Common Issues

🔴 "Cannot connect to ONI node"

Cause: The ONI P2P node is not running

Solution:

  1. Open the ONI Manager
  2. Click Start All or start individual components
  3. Verify with: curl http://127.0.0.1:6881/status

🔴 "Domain not found"

Cause: The domain hasn't been registered, or ONS is not running

Solution:

  1. Check ONS server is running: curl http://127.0.0.1:5353/status
  2. Register your domain through the ONI Manager or Registrar
  3. Wait a few seconds for the registration to propagate

🔴 "Port already in use"

Cause: Another application is using the same port

Solution:

  1. Find what's using the port: sudo lsof -i :6881
  2. Change the port in ONI Manager settings
  3. Or stop the conflicting application

🔴 "No peers found"

Cause: No other ONI nodes are reachable on your network

Solution:

  1. Ensure other ONI users are online
  2. Check firewall settings for incoming connections
  3. Try manual sync with a specific peer address
  4. Your node will still work for local browsing

🔴 "Docker container exits immediately"

Cause: Docker build issues or missing dependencies

Solution:

  1. Rebuild: docker-compose build --no-cache
  2. Check logs: docker-compose logs oni-node
  3. Ensure Dockerfile is in the correct directory

❓ Frequently Asked Questions

Q: Are .orb domains really free?

A: Yes! .orb domains are completely free to register with no renewal costs. They are managed by the ONI community and there's no central authority charging fees.

Q: Can I use my own domain (like .com) with ONI?

A: Currently, ONI only supports .orb domains. However, you can set up a gateway that bridges traditional DNS to the ONI network if you have technical expertise.

Q: Can I host dynamic websites with databases?

A: Yes! ONI serves standard HTML5, so you can use JavaScript to fetch data from APIs or databases. For dynamic backends, you'd need to run your own server alongside ONI and use JavaScript to connect to it.

Q: How is this different from a regular web host?

A: Regular web hosting relies on centralized servers. ONI is peer-to-peer — your content is hosted on your own machine or a server you control. There's no central company that can take down your site or charge you fees.

Q: Can I host multiple sites on one node?

A: Yes! You can host multiple .orb domains from a single ONI node. Just use --host-domain multiple times or register them through the ONI Manager.

Q: What happens if my computer goes offline?

A: Your hosted sites will be unavailable until you come back online. For always-on hosting, use a VPS or server with Docker deployment.

Q: Can I use HTTPS/SSL with ONI?

A: ONI uses its own security model with Ed25519 cryptographic signatures rather than traditional SSL/TLS. Content is cryptographically signed by the domain owner.

Q: How do I update my website content?

A: Simply update the files in your website directory and restart the ONI node hosting the domain: oni node --host-domain mysite.orb /path/to/mysite
